# Contributing / Contribution

## FR - Merci pour votre contribution

Les contributions sont les bienvenues: correction de bugs, ameliorations, documentation, tests.

### Workflow recommande

1. Forker le projet.
2. Creer une branche dediee (`feature/...` ou `fix/...`).
3. Faire des commits atomiques et explicites.
4. Ouvrir une Pull Request avec:
   - contexte
   - changements realises
   - etapes de test

### Bonnes pratiques

- Respecter le style de code existant.
- Eviter les regressions sur la moderation, le CSRF et l'anti-spam.
- Mettre a jour `CHANGELOG.md` pour tout changement notable.
- Ajouter/mettre a jour la documentation si necessaire.

## EN - Thanks for contributing

Contributions are welcome: bug fixes, enhancements, documentation, and tests.

### Recommended workflow

1. Fork the project.
2. Create a dedicated branch (`feature/...` or `fix/...`).
3. Make small, clear commits.
4. Open a Pull Request including:
   - context
   - implemented changes
   - test steps

### Good practices

- Follow the existing code style.
- Avoid regressions on moderation, CSRF, and anti-spam.
- Update `CHANGELOG.md` for notable changes.
- Update documentation when needed.
